# Feature Specification: 智能传媒内容分析与推荐系统

**Feature Branch**: `feature/smart-media-system`

**Created**: 2026-05-29

**Status**: Draft

**Input**: 构建智能传媒内容分析与推荐系统，面向短视频创作者提供热榜分析、AI企划生成、个性化推荐和点子库社区服务。团队4人，2周交付MVP。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 用户注册、登录与偏好设置 (Priority: P1)

作为一名短视频创作者，我需要注册账号、登录系统并设置我的创作领域偏好，以便系统为我提供个性化的热点推荐和内容企划服务。

**Why this priority**: 用户系统是所有个性化功能的基础，没有用户身份无法进行偏好匹配和推荐。

**Independent Test**: 可通过注册新账号、登录、设置创作领域偏好，验证全流程独立可运行。

**Acceptance Scenarios**:

1. **Given** 未注册用户访问系统，**When** 填写手机号/邮箱和密码完成注册，**Then** 系统创建账号并自动登录，跳转到偏好设置页。
2. **Given** 已注册用户输入正确凭据，**When** 点击登录，**Then** 系统验证身份并返回会话凭证，跳转到首页。
3. **Given** 已登录用户在个人中心，**When** 选择多个创作领域（如科技、美食、教育）并保存，**Then** 系统记录偏好，后续推荐基于此进行匹配。

---

### User Story 2 - 查看热榜与热点分析 (Priority: P1)

作为一名创作者，我需要浏览抖音等平台的热榜数据，并查看每个热点的关键词、分类、情感倾向和风险评级，以便判断哪些热点值得跟进创作。

**Why this priority**: 热榜是内容创作的源头数据，没有热点感知，后续推荐和企划都无法进行。

**Independent Test**: 打开热榜页面，验证热榜列表加载、筛选、点击热点查看分析结果的完整流程。

**Acceptance Scenarios**:

1. **Given** 用户在首页，**When** 进入热榜页面，**Then** 系统展示当前热榜列表，包含排名、标题、热度值和分类标签。
2. **Given** 热榜列表已加载，**When** 用户选择某个分类筛选或搜索关键词，**Then** 系统过滤展示匹配的热点列表。
3. **Given** 用户点击某个热点条目，**When** 进入热点详情页，**Then** 系统展示该热点的关键词提取、情感分析（正面/负面/中性）、风险评级（低/中/高）和内容分类。

---

### User Story 3 - 获取个性化选题推荐 (Priority: P1)

作为一名创作者，我希望系统根据我的创作领域和兴趣偏好，自动为我推荐匹配的热点选题，并给出推荐分数和理由。

**Why this priority**: 个性化推荐是系统的核心价值，直接决定创作者能否高效找到适合自己的选题。

**Independent Test**: 设置创作偏好后，访问推荐页面，验证推荐列表是否匹配偏好领域，每条推荐是否包含分数和理由。

**Acceptance Scenarios**:

1. **Given** 已登录且已设置创作偏好的用户，**When** 访问推荐页面，**Then** 系统展示与用户创作领域匹配的推荐选题列表，每条包含推荐分数和推荐理由。
2. **Given** 推荐列表已展示，**When** 用户对某条推荐点击"感兴趣"或"不感兴趣"，**Then** 系统记录反馈并调整后续推荐结果。

---

### User Story 4 - 生成内容企划案 (Priority: P1)

作为一名创作者，我选择一个热点并输入我的创作意图（想表达的观点、目标受众等），系统为我生成一份完整的内容企划案，包括标题建议、视频结构大纲、创新角度和风险评估。

**Why this priority**: AI企划生成是系统区别于普通热榜工具的核心竞争力，直接影响用户创作效率。

**Independent Test**: 选择一个热点，输入创作意图，验证生成的企划案包含标题建议、视频结构、拍摄建议和风险提示。

**Acceptance Scenarios**:

1. **Given** 用户在热点详情页或推荐页，**When** 选择一个热点并点击"生成企划"，**Then** 系统引导用户输入创作意图（观点角度、粉丝画像等可选字段）。
2. **Given** 用户已输入创作意图并确认，**When** 系统生成企划案，**Then** 展示包含以下内容的企划：3个标题建议、视频内容大纲（含开头吸引点）、拍摄和发布建议、风险提醒。
3. **Given** 企划案已生成，**When** 用户查看企划历史，**Then** 可回顾之前生成的所有企划案。

---

### User Story 5 - 浏览与发布点子 (Priority: P2)

作为一名创作者，我希望在点子库社区中浏览其他创作者分享的创意点子，也可以发布自己的创意，与其他创作者交流碰撞。

**Why this priority**: 点子库增强用户粘性和社区活跃度，但在MVP中可作为P1功能完成后的增值模块。

**Independent Test**: 进入点子库页面，验证可浏览点子列表、筛选分类、发布新点子。

**Acceptance Scenarios**:

1. **Given** 已登录用户，**When** 进入点子库页面，**Then** 展示按热度排序的点子列表，支持按分类筛选。
2. **Given** 用户在点子库页面，**When** 点击"发布点子"，填写标题、内容、关联热点和分类后提交，**Then** 点子发布成功并显示在列表中。
3. **Given** 用户在点子详情页，**When** 对其他创作者的点子点赞或评论，**Then** 互动行为记录并更新显示。

---

### User Story 6 - 点子互动与热门排序 (Priority: P2)

作为一名创作者，我希望对点子库中的内容进行点赞和评论互动，系统根据热度对点子进行排序展示。

**Why this priority**: 互动机制是社区的核心驱动，与点子发布配套形成完整社区体验。

**Independent Test**: 验证点赞/评论功能正常，热门排序规则符合预期。

**Acceptance Scenarios**:

1. **Given** 用户浏览点子列表，**When** 点击某条点子的"点赞"按钮，**Then** 点赞数+1，按钮状态变为已点赞。
2. **Given** 用户查看某条点子详情，**When** 提交评论内容，**Then** 评论出现在评论区，其他用户可见。
3. **Given** 点子库中有多条点子，**When** 按"热门"排序，**Then** 系统根据综合热度（点赞数+评论数+发布时间衰减）排序展示。

---

### User Story 7 - 管理员内容审核 (Priority: P3)

作为运营管理员，我需要审核用户发布的点子内容，确保平台内容合规。

**Why this priority**: 内容审核是平台合规运营的必需功能，但在MVP阶段可简化为基础审核流程。

**Independent Test**: 管理员账号登录后，查看待审核内容列表，执行通过/拒绝操作。

**Acceptance Scenarios**:

1. **Given** 管理员登录系统，**When** 进入审核页面，**Then** 展示所有待审核的点子列表。
2. **Given** 管理员查看某条待审核内容，**When** 点击"通过"，**Then** 内容发布成功并对所有用户可见。
3. **Given** 管理员查看某条内容，**When** 点击"拒绝"并填写拒绝原因，**Then** 内容不公开且作者收到拒绝通知。

---

### Edge Cases

- 热榜数据源不可用时，系统如何处理？（展示最近一次缓存数据并提示"数据更新中"）
- 用户未设置创作偏好时，推荐如何处理？（推荐通用热门选题，引导用户完成偏好设置）
- AI模型调用超时或失败时，企划生成如何处理？（返回错误提示，允许用户重试）
- 同一用户对同一热点多次生成企划，系统如何处理？（允许重新生成，保留历史版本）
- 并发用户超过100时，系统如何处理？（排队机制或降级为静态热榜展示）
- 用户输入创作意图为空时，企划生成如何处理？（使用默认模板生成通用企划案）
- 点子被管理员拒绝后，作者是否可以编辑重发？（支持修改后重新提交审核）
- 热榜数据更新过程中用户正在查看，如何保证数据一致性？（使用缓存快照，更新完成后切换）

## Requirements *(mandatory)*

### Functional Requirements

**用户系统**
- **FR-001**: 系统 MUST 支持用户通过手机号或邮箱注册账号，密码经过安全加密存储。
- **FR-002**: 系统 MUST 提供登录功能，验证通过后发放有时效限制的会话凭证。
- **FR-003**: 用户 MUST 能够在个人中心设置创作领域偏好（多选）和兴趣标签。
- **FR-004**: 用户 MUST 能够查看和编辑个人资料信息。

**热榜数据采集与展示**
- **FR-005**: 系统 MUST 定时自动采集外部平台热榜数据（每10分钟一次），存储到持久化存储。
- **FR-006**: 系统 MUST 展示热榜列表，包含排名、标题、热度值、分类标签，至少展示前20条。
- **FR-007**: 系统 MUST 支持按分类、热度、时间多维度筛选和排序热榜数据。
- **FR-008**: 系统 MUST 支持对热榜数据进行关键词搜索。

**热点数据分析**
- **FR-009**: 系统 MUST 对采集的热点进行自动分析，提取关键词和主题。
- **FR-010**: 系统 MUST 对每个热点进行内容分类（至少8个类别：时政、财经、科技、娱乐、体育、美食、教育、其他）。
- **FR-011**: 系统 MUST 对热点进行情感倾向分析（正面/负面/中性）。
- **FR-012**: 系统 MUST 对热点进行风险评级（低/中/高），供创作者规避风险。

**个性化推荐**
- **FR-013**: 系统 MUST 根据用户创作偏好和热点特征匹配生成推荐选题列表。
- **FR-014**: 系统 MUST 为每条推荐结果提供推荐分数和推荐理由。
- **FR-015**: 系统 MUST 支持用户对推荐结果进行反馈（感兴趣/不感兴趣），并据此优化后续推荐。
- **FR-016**: 系统 MUST 支持A/B测试框架，可对比不同推荐策略的效果。

**内容企划生成**
- **FR-017**: 系统 MUST 支持用户选择热点并生成内容企划案。
- **FR-018**: 系统 MUST 在生成企划前允许用户输入创作意图（包括观点角度、目标受众描述）。
- **FR-019**: 系统 MUST 生成的企划案包含：至少3个标题建议、视频内容大纲、开头吸引点、拍摄建议、风险提醒。
- **FR-020**: 用户 MUST 能够查看已生成的历史企划列表。
- **FR-021**: 系统 MUST 支持预定义的企划模板，适应不同视频类型需求。

**点子库社区**
- **FR-022**: 用户 MUST 能够在点子库中发布创意点子（标题、正文、关联热点、分类标签）。
- **FR-023**: 用户 MUST 能够浏览所有已发布的点子列表，支持按分类筛选和热门排序。
- **FR-024**: 用户 MUST 能够对其他用户的点子进行点赞。
- **FR-025**: 用户 MUST 能够对其他用户的点子发表评论。
- **FR-026**: 系统的热门排序 MUST 综合点赞数、评论数和发布时间进行加权计算。

**管理功能**
- **FR-027**: 管理员 MUST 能够审核用户发布的内容，支持通过或拒绝操作并填写原因。
- **FR-028**: 管理员 MUST 能够查看系统运营数据统计（用户量、热点量、推荐量、点子量）。

### Key Entities

- **用户（User）**: 系统使用者。包含账号信息、密码凭证、创作偏好标签、个人资料。创作者是核心用户角色，管理员具有审核和统计权限。
- **热点（HotTopic）**: 从外部平台采集的热门内容条目。包含标题、排名、热度值、来源分类、采集时间。一个热点经过分析后关联多个分析结果。
- **热点分析结果（TopicAnalysis）**: 对热点的NLP分析输出。包含关键词列表、内容分类、情感倾向、风险评级。与热点一一对应。
- **推荐记录（Recommendation）**: 系统为用户生成的选题推荐。包含推荐分数、推荐理由、关联的热点和用户，记录用户反馈（感兴趣/不感兴趣）。
- **内容企划（ContentPlan）**: AI生成的创作企划案。包含标题建议列表、视频大纲、拍摄建议、风险提示。关联用户、热点和创作意图输入。
- **创作意图（CreationIntent）**: 用户输入的自定义创作方向。包含观点角度、目标受众描述。作为企划生成的输入参数。
- **点子（Idea）**: 用户在社区分享的创意内容。包含标题、正文、关联热点、分类标签、发布者、发布时间、审核状态。
- **点子互动（IdeaInteraction）**: 用户对点子的互动行为。包含点赞和评论。评论支持嵌套回复。
- **审核记录（ReviewRecord）**: 管理员的内容审核记录。包含审核人、审核结果、拒绝原因、审核时间。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可在2分钟内完成从注册到查看个性化推荐的全流程。
- **SC-002**: 热榜页面在2秒内完成数据加载和渲染（数据量约50条）。
- **SC-003**: 内容企划从用户确认生成到展示完整结果不超过30秒。
- **SC-004**: 系统支持100名用户同时在线使用，核心功能无明显延迟或报错。
- **SC-005**: 推荐准确率方面，至少60%的用户反馈为"感兴趣"（基于推荐反馈功能收集）。
- **SC-006**: 系统核心功能（注册登录、热榜浏览、推荐展示、企划生成、点子浏览）在2周内完成开发并通过验收测试，核心功能测试通过率不低于95%。
- **SC-007**: 用户可在首次使用时无需指导即完成"注册→查看热榜→获取推荐→生成企划"的完整操作链路。
- **SC-008**: 点子库用户可在30秒内完成一条点子的发布流程。

## Assumptions

- 热榜数据源在项目期间可用，如外部接口不可用，使用预采集的模拟数据完成演示。
- AI大模型API在项目期间可正常调用，如不可用，系统支持使用预设模板生成企划内容作为降级方案。
- 项目部署环境为单机或小规模云服务器，暂不考虑大规模分布式部署。
- 用户手机号/邮箱注册为简化验证（演示用），不接入短信网关和邮件服务。
- 前端支持桌面端现代浏览器（Chrome、Firefox、Edge最近两个版本），移动端适配为可选需求。
- 系统面向中文内容创作者，界面和内容均为中文。
- 团队成员均按计划投入时间，无重大请假或不可抗力影响。
